# QuickDrop

Windows向けのシンプルなFTPサーバアプリケーションです。GUIウィンドウでログを確認しながら、手軽にFTPサーバを起動・停止できます。

## 特徴

- **GUIインターフェース**: 直感的な操作画面
- **リアルタイムログ表示**: 接続状況やファイル転送をリアルタイムで確認
- **ID/Password認証**: セキュアなアクセス制御
- **ポート番号カスタマイズ**: 任意のポート番号を設定可能
- **ディレクトリ選択**: GUIから簡単にルートディレクトリを選択
- **ワンクリック起動/停止**: ボタンひとつでサーバの起動・停止
- **Windows最適化**: UTF-8文字コード対応、Windowsパス処理
- **日本語対応**: メイリオフォントで日本語を正しく表示

## 必要要件

- Windows 10/11

## インストール

### ビルド済み実行ファイル

`target/release/quickdrop.exe` を好きな場所にコピーして使用できます。

### ソースからビルド

```bash
cargo build --release
```

ビルドされた実行ファイルは `target/release/quickdrop.exe` に生成されます。

## 使用方法

### 1. アプリケーションの起動

`quickdrop.exe` をダブルクリックするだけで起動します。
コンソールウィンドウは表示されません。

```bash
# またはコマンドラインから
cargo run --release
./target/release/quickdrop.exe
```

### 2. 設定

GUIウィンドウが開いたら、以下の項目を設定できます：

- **ポート**: FTPサーバが待ち受けるポート番号（デフォルト: 2121）
- **ユーザー名**: FTP接続時のユーザー名（デフォルト: admin）
- **パスワード**: FTP接続時のパスワード（デフォルト: password）
- **ルートディレクトリ**: FTPで公開するディレクトリ
  - テキスト入力で直接指定
  - 「参照...」ボタンでフォルダダイアログから選択

### 3. サーバの起動

1. 設定を確認・変更
2. 「▶ 起動」ボタンをクリック
3. ログエリアに起動メッセージが表示されます
4. ステータスが「[稼働中]」に変わります

**注意**: サーバ起動中は設定を変更できません。

### 4. FTPクライアントで接続

#### FileZillaでの接続

- ホスト: `localhost` または `127.0.0.1`
- ポート: 設定したポート番号（例: 2121）
- プロトコル: FTP
- ユーザー名: 設定したユーザー名
- パスワード: 設定したパスワード

#### コマンドラインでの接続

```cmd
ftp localhost 2121
```

### 5. サーバの停止

「■ 停止」ボタンをクリックすると、サーバが停止します。

### 6. アプリケーションの終了

ウィンドウの「×」ボタンをクリックすると、自動的にサーバが停止してアプリケーションが終了します。

## ログの見方

ログエリアには以下の情報がリアルタイムで表示されます：

- サーバの起動/停止メッセージ
- クライアントの接続/切断
- 認証の成功/失敗
- FTPコマンドの実行状況
- ファイルのアップロード/ダウンロード完了

ログは自動的に最下部にスクロールされます。

## 設定ファイル

初回起動時、または「▶ 起動」ボタンを押すと、現在の設定が `config.toml` に保存されます。
次回起動時はこの設定が自動的に読み込まれます。

```toml
port = 2121
username = "admin"
password = "password"
root_dir = "./ftp_root"
```

手動で編集することも可能です。

## 対応FTPコマンド

- `USER`/`PASS`: 認証
- `LIST`: ファイル一覧
- `RETR`: ファイルダウンロード
- `STOR`: ファイルアップロード
- `PWD`, `CWD`: ディレクトリ操作
- `MKD`/`XMKD`: ディレクトリ作成
- `RMD`/`XRMD`: ディレクトリ削除
- `DELE`: ファイル削除
- `PASV`: パッシブモード
- その他: `SYST`, `TYPE`, `QUIT`, `NOOP`

## 使用例

### ローカルPC間でのファイル共有

1. 送信側PCでQuickDropを起動
2. 共有したいフォルダをルートディレクトリに設定
3. サーバを起動
4. 受信側PCからFTPクライアントで接続
5. ファイルをダウンロード
6. 完了したらサーバを停止

### 一時的なファイル転送

1. QuickDropを起動
2. デフォルト設定のまま「▶ 起動」
3. `ftp_root` フォルダにファイルを配置
4. 他のPCから接続してダウンロード
5. 「×」ボタンでアプリを閉じる

## セキュリティに関する注意

- **開発・テスト用途**: このサーバは学習や開発、一時的なファイル転送用です
- **平文パスワード**: パスワードは暗号化されずに保存されます
- **ローカルネットワーク推奨**: 信頼できるネットワーク内でのみ使用してください
- **機密データに注意**: 重要なデータの公開には使用しないでください
- **ファイアウォール**: 必要に応じてファイアウォール設定を確認してください

## トラブルシューティング

### ポートが既に使用されている

- 別のポート番号（例: 2122, 3000など）を試してください
- 他のFTPサーバが起動していないか確認してください

### ルートディレクトリにアクセスできない

- ディレクトリパスが正しいか確認してください
- 書き込み権限があるディレクトリを指定してください
- 「参照...」ボタンから改めて選択してください

### 接続できない

- サーバが起動しているか確認してください（[稼働中]表示）
- ポート番号が正しいか確認してください
- Windowsファイアウォールの設定を確認してください
- ログエリアにエラーメッセージが表示されていないか確認してください

### 日本語が文字化けする

- Windows 10/11のメイリオフォントが正しくインストールされているか確認してください
- フォントファイルが `C:\Windows\Fonts\meiryo.ttc` に存在することを確認してください

## プロジェクト構成

```
D:\scripts\rust\local-ftp\
├── Cargo.toml              # 依存関係定義
├── build.rs                # ビルドスクリプト（アイコン設定）
├── quick_drop.ico          # アプリケーションアイコン
├── config.toml             # サーバ設定ファイル（自動生成）
├── README.md               # このファイル
├── src\
│   ├── main.rs             # GUIアプリケーション
│   └── ftp_server.rs       # FTPサーバコア実装
├── ftp_root\               # FTP公開ディレクトリ（自動生成）
└── target\release\
    └── quickdrop.exe       # ビルド済み実行ファイル
```
